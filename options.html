Privacy & Transparency

Switcheroo is built with your privacy as the top priority. Here's exactly what this extension does and doesn't do:

  • No browsing data collected. Switcheroo uses the browser's Declarative Net Request API, which handles redirects at the browser level. The extension code never sees which YouTube videos or Reddit posts you visit.
  • No analytics or telemetry. Zero usage data is collected or transmitted.
  • Local storage only. Your settings are stored in chrome.storage.local — not synced to any server, not uploaded to Google.
  • Network requests are made to fetch up-to-date instance lists. By default this happens every hour; you can change or disable this entirely in the Instance Lists card at the top of this page. The requests go to:
    • https://api.invidious.io/instances.json — YouTube (Invidious)
    • https://raw.githubusercontent.com/redlib-org/redlib-instances/…/instances.json — Reddit (Redlib)
    • https://rimgo.codeberg.page/api.json — Imgur (Rimgo)
    • https://raw.githubusercontent.com/pablouser1/ProxiTok/…/instances.json — TikTok (ProxiTok)
    • https://git.sr.ht/~edwardloveall/scribe/blob/main/docs/instances.md — Medium (Scribe)
    Each request reveals to that server that you have Switcheroo installed (your IP address and User-Agent are visible). No browsing history or redirect activity is included. Set the auto-update interval to Off to make zero automatic network calls — you can still refresh each list manually at any time.
  • Cloudflare instances excluded by default. Cloudflare acts as a middleman that can log traffic, defeating the purpose of a privacy frontend. Cloudflare-backed instances are shown separately and disabled by default.
  • HTTPS only. Only HTTPS instances are ever used as redirect targets. HTTP instances are rejected automatically.
